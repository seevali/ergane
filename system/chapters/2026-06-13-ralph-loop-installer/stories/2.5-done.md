## Key Implementation Decisions

### `hashFile` reuse
`doctor.js` imports `hashFile` from `writer.js` rather than re-implementing it. This ensures the checksum algorithm is identical between install and validation, so unmodified files always pass the checksum check.

### Injectable `checkCommand`
`runDoctor` accepts `opts.checkCommand` to replace the real `which`-based check for unit testing. This follows the same injection pattern as `preflight.js`'s `opts.commandExists`. All 26 doctor tests use this injection to avoid shelling out to `which` in tests.

### `GETTING-STARTED.md` ownership
`GETTING-STARTED.md` is not listed in `getOwnership()`'s installer-owned list (which only covers `scripts/ralph-loop.sh`, `scripts/prompts/**`, `docs/project-conventions.md`, `.gitignore`). Therefore it falls through to `user-owned` automatically — no changes to `getOwnership()` were needed.

### Doctor does not call `process.exit`
`runDoctor` returns `{ passed, findings }` and never calls `exit`. Exit codes 0/1/2 are handled entirely by `bin/ralph.js`'s `doctor` command action handler. This makes unit testing straightforward (no `opts.exit` injection needed in tests).

### Epic header parsing
The doctor only emits an `epic-headers` finding when `### Story` text exists in the file but the `### Story X.Y:` regex finds zero matches (malformed headers). Files with zero `### Story` lines at all are silently skipped — this handles new projects that haven't added story headers yet.

## Code Review Fixes (2026-06-13)

**Review verdict was BLOCK** on one issue: two AC-2 behaviours were dead in production because `outro.js` read fields that the real `InstallPlan`/manifest never set.

### Defect 1 — existing-project warning (AC-2)
- **Root cause:** `outro.js` checked `plan.targetClass === 'existing-project'`, but the real wizard sets `plan.classification`. `plan.targetClass` was always `undefined`, so the warning never fired.
- **Fix:** Changed to `plan.classification === 'existing-project'` in `outro.js:31`.

### Defect 2 — BMAD memory step hidden when skipBmad (AC-2)
- **Root cause:** `outro.js` checked `result.manifest?.wizardAnswers?.skipBmad`. The real wizard stores `skipBmad` as a top-level plan field, not inside `wizardAnswers`. So the guard was always `undefined !== true → true`, and step 5 always printed.
- **Fix:** Changed to `plan.skipBmad !== true` in `outro.js:25`.

### Test fixes
- `makePlan()` default changed from `targetClass: 'empty'` to `classification: 'empty'`.
- skipBmad tests updated to pass `makePlan({ skipBmad: true/false })` instead of fabricating `result.manifest.wizardAnswers.skipBmad`.
- Existing-project tests updated to use `classification: 'existing-project'` in the plan.

Post-fix: 151/151 tests pass, checkpoint clean.
