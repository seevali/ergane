## Key Implementation Decisions

### Package name: `@seevali/ralph-loop`

Both candidates were checked against npm:
- `npm view @seevali/ralph-loop` → 404 (available)
- `npm view ralph-loop-installer` → 404 (available)

Per the decision logic in the spec, `@seevali/ralph-loop` was chosen because scoped names are preferred for maintainer-specific packages and reduce squatting risk.

### `installer/LICENSE` copy

The `LICENSE` file lives at the repo root. npm's `files` array in `installer/package.json` resolves paths relative to the package directory (`installer/`). Since the repo root LICENSE is outside that scope, npm pack would silently omit it. The fix: copy the LICENSE to `installer/LICENSE`. This keeps the authoritative file at the repo root while ensuring the published tarball is correctly licensed.

### `files` array: `scripts/` removed

The prior `files` array included `installer/scripts/` (which contains `sync-templates.sh`, a dev utility). This script is not needed at runtime by end users and was removed from the published footprint. The finalized array is: `bin/`, `src/`, `templates/`, `README.md`, `LICENSE`.

### `installer/README.md`: created (was missing)

No README existed in `installer/`. Created a short one (18 lines) suitable for npm.org display: quick-start command, feature list, prerequisites, link to main README.

### `PUBLISHING.md`: complete manual publish checklist

Written from scratch at the repo root. Covers:
- Prerequisites (Node, npm account, 2FA, git state)
- Pre-publish checklist (tests, doctor smoke test, pack size, git state)
- Publish workflow (login → publish → verify → cold-start test)
- Rollback (< 24 h: unpublish; ≥ 24 h: deprecate + bump version)
- Appendix: full metadata reference for the published package
