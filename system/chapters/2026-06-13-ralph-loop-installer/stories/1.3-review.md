## Non-blocking observations (do NOT block; recorded for later stories)

1. **`check_mode` does not detect stale files that exist only in the destination.** It iterates source files (`find "$SOURCE_PROMPTS" -type f`) and compares each to its dest counterpart, so a file present in `installer/templates/loop/prompts/` but absent from `scripts/prompts/` is invisible to `--check` (verified: a planted `STALE_EXTRA.md` left `--check` at exit 0). This is consistent with the AC wording ("compare each synced file against its source") and is self-corrected by `sync_mode`'s `rm -rf "$DEST_PROMPTS"` on every sync, so drift cannot persist through a resync. If the manifest work in a later story needs to gate on deletions too, consider adding a reverse pass (iterate dest, flag files with no source). Out of scope for 1.3.

2. **One non-reproducible transient.** During the first batch run, a single subdir `--check` returned exit 1 with no stderr; it did not recur across 10 subsequent clean runs or any later invocation. Most likely a WSL2 filesystem metadata-sync hiccup immediately following the preceding `rm -rf`+`cp -rp` resync, not a logic defect (every failure path in the script emits a message, and this produced none). Noted only for awareness; nothing actionable.

No cross-story root-cause issues found — the script consumes existing `scripts/ralph-loop.sh` and `scripts/prompts/**` as read-only inputs and required no upstream changes.
