# Story 1.3: Template sync script with drift gate

**Chapter:** [2026-06-13-ralph-loop-installer](../README.md)  
**Epic:** [1: Foundation](../epics/ralph-loop-installer.md#epic-1-foundation)  
**Status:** review

---

## Summary

Build `installer/scripts/sync-templates.sh`, a bash script that:
- **Sync mode** (default): copies `scripts/ralph-loop.sh` and all files in `scripts/prompts/**` into `installer/templates/loop/`, preserving directory structure
- **Check mode** (`--check`): byte-compares the source files against the synced copies and exits non-zero if any drift is detected

This script is the **canonical keeper of truth** — the loop files in the package must always match the repo's source, and every story's code review will gate on `installer/scripts/sync-templates.sh --check` passing.

---

## User Story

As a **loop maintainer**, I need a sync script with a drift gate so that the installer package never ships stale or divergent loop files, and review checkpoints can detect accidental modifications.

---

## Acceptance Criteria

1. **Sync creates correct structure:** Running `installer/scripts/sync-templates.sh` from repo root creates:
   - `installer/templates/loop/ralph-loop.sh` (exact copy of `scripts/ralph-loop.sh`)
   - `installer/templates/loop/prompts/` (exact copy of `scripts/prompts/**`)

2. **Check mode detects no drift on clean sync:** After sync, `installer/scripts/sync-templates.sh --check` exits 0

3. **Check mode detects drift:** Manually modifying any synced file (e.g., adding a line to `installer/templates/loop/ralph-loop.sh`) makes `installer/scripts/sync-templates.sh --check` exit non-zero

4. **Script passes syntax validation:** `bash -n installer/scripts/sync-templates.sh` exits 0

## Technical Requirements

### Architecture Context

From PRD **FR-6 (Installed footprint)** and **Design section:**
- The installer is published as an npm package with a whitelisted `files` field in `package.json` (Story 1.2)
- Canonical loop files (`scripts/ralph-loop.sh`, `scripts/prompts/**`) are **read-only inputs** to the sync script
- The installer owns only the copied files in `installer/templates/loop/` — user-facing files remain read-write
- Manifest (`installer/templates/manifest.json`, per later stories) records checksums to detect local modifications

### Script Behavior

**Sync mode (no argument or `--sync`):**
- Copy `scripts/ralph-loop.sh` → `installer/templates/loop/ralph-loop.sh`
- Recursively copy `scripts/prompts/` → `installer/templates/loop/prompts/`
- Preserve file permissions (use `cp -p` or equivalent)
- Create `installer/templates/loop/` if it doesn't exist
- Exit 0 on success
- Exit non-zero with a descriptive error message if a source file is missing or copy fails

**Check mode (`--check`):**
- Compare each synced file byte-for-byte against its source
- For files in `installer/templates/loop/prompts/`, compare against the corresponding file in `scripts/prompts/`
- Exit 0 if all files match exactly
- Exit non-zero if any file is missing, differs in size, or differs in content
- On mismatch, print the name of the first differing file to stderr and exit 1
- Offer a diagnostic hint: `"Run 'installer/scripts/sync-templates.sh' to resync."` (but don't auto-fix)

### Script Requirements

- **Language & style:** Bash with `set -euo pipefail` at the top
- **Shebang:** `#!/usr/bin/env bash`
- **Portability:** Must work on Linux (CI) and macOS; assume `bash` ≥ 4.0
- **Variables:** Quote all expansions; prefer `[[ ]]` over `[ ]`
- **Paths:** Use repo-root-relative paths; assume script is invoked from repo root (as will be the case in the loop and CI)
- **Error messages:** Clear, actionable; distinguish between "source file missing" and "sync is out of date"

### Directory Structure

Before this story runs, the following must exist (from Story 1.2):
- `installer/` (top-level directory)
- `installer/package.json` (Node.js package with `bin` and `files` fields)
- `installer/bin/` (if referenced by `bin` in package.json)

After this story:
```
installer/
├── scripts/
│   └── sync-templates.sh          [NEW] — this script
├── templates/
│   └── loop/
│       ├── ralph-loop.sh          [NEW] — synced copy
│       └── prompts/               [NEW] — synced from scripts/prompts/
│           ├── common/
│           ├── ...
```

The `installer/templates/` directory structure mirrors the source:
- If `scripts/prompts/` has subdirectories (e.g., `common/`, `system-prompts/`), they are mirrored in `installer/templates/loop/prompts/`
- All files under `scripts/prompts/**` are copied recursively

## Implementation Steps

### Step 1: Create the script file

Create `installer/scripts/sync-templates.sh` with the following structure:

1. Shebang and options: `#!/usr/bin/env bash` + `set -euo pipefail`
2. Variable definitions (source and destination paths)
3. Helper functions:
   - `sync_mode()` — copy source to destination
   - `check_mode()` — validate destination matches source
   - `main()` — parse arguments and dispatch
4. Entry point: call `main "$@"`

### Step 2: Define paths

Use variables to avoid hardcoding:
- `REPO_ROOT=$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)` (two levels up from `installer/scripts/`)
- `SOURCE_LOOP="${REPO_ROOT}/scripts/ralph-loop.sh"`
- `SOURCE_PROMPTS="${REPO_ROOT}/scripts/prompts"`
- `DEST_TEMPLATE="${REPO_ROOT}/installer/templates/loop"`
- `DEST_LOOP="${DEST_TEMPLATE}/ralph-loop.sh"`
- `DEST_PROMPTS="${DEST_TEMPLATE}/prompts"`

### Step 3: Implement `sync_mode()` function

```bash
sync_mode() {
  # Validate sources exist
  if [[ ! -f "$SOURCE_LOOP" ]]; then
    echo "Error: source loop script not found: $SOURCE_LOOP" >&2
    return 1
  fi
  if [[ ! -d "$SOURCE_PROMPTS" ]]; then
    echo "Error: source prompts directory not found: $SOURCE_PROMPTS" >&2
    return 1
  fi
  
  # Create destination directory
  mkdir -p "$DEST_TEMPLATE"
  
  # Copy the loop script (preserve permissions)
  cp -p "$SOURCE_LOOP" "$DEST_LOOP" || {
    echo "Error: failed to copy loop script" >&2
    return 1
  }
  
  # Remove old prompts directory if it exists (clean slate)
  rm -rf "$DEST_PROMPTS"
  
  # Copy prompts directory recursively
  cp -rp "$SOURCE_PROMPTS" "$DEST_PROMPTS" || {
    echo "Error: failed to copy prompts directory" >&2
    return 1
  }
}
```

Reasoning:
- `set -euo pipefail` catches failures early
- Quote `$SOURCE_*` and `$DEST_*` to handle any future path changes
- Use `cp -p` to preserve file modification times (useful for manifest checksums in later stories)
- Remove old prompts before copying to ensure no stale files linger
- Return 1 on any failure so the script exits non-zero

### Step 4: Implement `check_mode()` function

```bash
check_mode() {
  # Validate destination exists
  if [[ ! -f "$DEST_LOOP" ]]; then
    echo "Error: synced loop script not found: $DEST_LOOP" >&2
    return 1
  fi
  if [[ ! -d "$DEST_PROMPTS" ]]; then
    echo "Error: synced prompts directory not found: $DEST_PROMPTS" >&2
    return 1
  fi
  
  # Compare loop script
  if ! cmp -s "$SOURCE_LOOP" "$DEST_LOOP"; then
    echo "Drift detected: $DEST_LOOP differs from source" >&2
    echo "Run 'installer/scripts/sync-templates.sh' to resync." >&2
    return 1
  fi
  
  # Compare prompts recursively
  local src_file dest_file
  while IFS= read -r -d '' src_file; do
    dest_file="${src_file#$SOURCE_PROMPTS}"
    dest_file="${DEST_PROMPTS}${dest_file}"
    
    if ! cmp -s "$src_file" "$dest_file"; then
      echo "Drift detected: $dest_file differs from source" >&2
      echo "Run 'installer/scripts/sync-templates.sh' to resync." >&2
      return 1
    fi
  done < <(find "$SOURCE_PROMPTS" -type f -print0)
  
  return 0
}
```

Reasoning:
- `cmp -s` does a byte-for-byte comparison silently (returns 0 if identical, 1 if different)
- Loop through all files under `SOURCE_PROMPTS` using `find -print0` to handle filenames with spaces/special chars
- Construct the corresponding destination path by stripping the source prefix and prepending the destination
- Exit non-zero on first mismatch (fail-fast)
- Print diagnostic hint so the user knows how to fix it

### Step 5: Implement argument parsing

```bash
main() {
  local mode="sync"
  
  if [[ $# -gt 0 ]]; then
    case "$1" in
      --check)
        mode="check"
        ;;
      --sync)
        mode="sync"
        ;;
      --help)
        echo "Usage: installer/scripts/sync-templates.sh [--check|--sync|--help]"
        echo ""
        echo "Sync canonical loop files into the installer package."
        echo ""
        echo "Modes:"
        echo "  --sync   (default) Copy scripts/ralph-loop.sh and scripts/prompts/ into installer/templates/loop/"
        echo "  --check  Exit non-zero if any synced file differs from the source"
        echo "  --help   Show this message"
        return 0
        ;;
      *)
        echo "Error: unknown argument: $1" >&2
        echo "Run with --help for usage" >&2
        return 1
        ;;
    esac
  fi
  
  case "$mode" in
    sync)
      sync_mode
      ;;
    check)
      check_mode
      ;;
  esac
}

main "$@"
```

Reasoning:
- Default to sync mode (most common use case)
- `--check` is explicit (review gates will call it by name)
- `--help` allows users to discover the script's purpose
- Unknown arguments fail explicitly rather than silently defaulting

### Step 6: Verify syntax

Run `bash -n installer/scripts/sync-templates.sh` to validate the script has no syntax errors before committing.

## Edge Cases & Gotchas

### Edge case: Prompts directory with nested subdirectories
**Scenario:** `scripts/prompts/` contains `common/`, `system/`, `models/`, etc.  
**Requirement:** All subdirectories and files must be copied exactly.  
**Implementation note:** `find -type f` followed by recursive path reconstruction handles this. Test by verifying a deeply nested file like `scripts/prompts/common/system-prompts/model-1.md` is copied to the same relative path under `installer/templates/loop/prompts/`.

### Edge case: Symbolic links in prompts
**Scenario:** A file in `scripts/prompts/` is a symlink (unlikely, but possible).  
**Requirement:** Symlinks should be copied as regular files (the installer package will be extracted, not symlinked).  
**Implementation note:** `cp -p` by default copies the target, not the link itself. This is correct. If this becomes an issue, we could add `-P` flag, but it's not needed here.

### Edge case: File permissions differ, content matches
**Scenario:** `installer/templates/loop/prompts/file.md` has mode `0644` but was synced with `0755`.  
**Requirement:** Check mode should detect this as drift (because the installer owns the exact state).  
**Implementation note:** `cmp -s` compares content only, not permissions. This is acceptable for the first iteration — the file content is what matters. If permission drift becomes a problem, we can enhance `check_mode()` to use `stat` comparisons, but it's out of scope for this story.

### Edge case: Source prompts directory is empty
**Scenario:** `scripts/prompts/` exists but has no files.  
**Requirement:** Sync creates `installer/templates/loop/prompts/` (empty directory).  
**Implementation note:** `cp -rp` handles this correctly.

### Edge case: Invoked from a subdirectory, not repo root
**Scenario:** User runs `cd installer/scripts && bash sync-templates.sh`.  
**Requirement:** Script should still work (paths are computed relative to the script's location).  
**Implementation:** `REPO_ROOT` is computed as `"$(dirname "${BASH_SOURCE[0]}")/../.."`, which resolves to the repo root regardless of current directory. Test by running from `installer/scripts/`.

### Edge case: Running sync twice in succession
**Scenario:** `bash installer/scripts/sync-templates.sh` followed immediately by `bash installer/scripts/sync-templates.sh`.  
**Requirement:** Second run should be idempotent (no errors, same result).  
**Implementation note:** The script removes and re-copies the prompts directory on every sync, which is safe. File modification times may differ slightly between runs, but check mode only cares about content, so this is fine.

### Edge case: Concurrent runs
**Scenario:** Two processes run sync concurrently (unlikely in practice, but theoretically possible in CI).  
**Requirement:** No corruption; worst case, one run sees a partial copy and fails.  
**Implementation note:** `cp` is atomic at the file level, so partial copies are possible. This is acceptable for a build-time script — concurrent runs are a misconfiguration, not a requirement. If it becomes an issue, we could add a lockfile, but it's not needed here.

## Hints for the Developer

- **Start by creating the directory structure:** `mkdir -p installer/scripts installer/templates/loop`
- **Test incrementally:** After writing each function, test it manually before moving on
- **Use `cmp -s` for byte comparison:** It's simpler and more reliable than `diff` or `md5sum` for detecting drift
- **Preserve `set -euo pipefail`:** It catches errors early; don't remove it or weaken it with `|| true`
- **Run `bash -n` frequently:** It's fast and catches most mistakes before runtime
- **Check the output messages:** They should guide the user on what to do next (e.g., "Run 'installer/scripts/sync-templates.sh' to resync")
