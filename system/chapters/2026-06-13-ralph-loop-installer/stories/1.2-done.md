## Key Implementation Decisions

1. **No deviation from the story spec.** Every file matches the story specification exactly — package.json structure, Commander flag strings, stub messages, and test assertion.

2. **`"type": "module"` and ESM imports.** The `bin/ralph.js` uses `import` (not `require`). The story spec required this; it aligns with modern Node.js (v20+) and avoids a CommonJS/ESM interop cliff in later stories.

3. **Version read from `package.json` at runtime.** Rather than hardcoding `"0.1.0"` in `ralph.js`, the script reads `package.json` via `readFileSync`. Single source of truth — bumping the version in `package.json` propagates automatically.

4. **`node:test` with no external test runner.** The `devDependencies` section is deliberately empty. Node 20's built-in test module handles the placeholder test with no extra packages, keeping the install lean.

5. **Empty placeholder directories are not tracked by git.** `src/`, `templates/`, and `scripts/` contain no `.gitkeep` files — they'll appear once stories populate them, consistent with git's "track content, not directories" model.

## Review Fix (post-review)

**Issue:** AC #4 (`npm run lint` passes on all JS files) was failing. The original lint script was:
```json
"lint": "bash -n bin/ralph.js && bash -n scripts/*.sh"
```
Two defects: (1) `bash -n` runs the Bash parser against a JS file — JS is not shell syntax, so it exits non-zero. (2) `scripts/*.sh` globs against an empty directory and errors.

**Fix applied to `installer/package.json:21`:**
```json
"lint": "node --check bin/ralph.js && node --check test/placeholder.test.js"
```
Uses `node --check` (Node's built-in syntax checker) on both JS files. No shell files exist yet in `scripts/`; they can be added to the lint command in the story that creates them.
